# 教师端班级学习分析 PRD

## 背景

当前后端已有 `GET /api/analytics/overview` 和学生端 `GET /api/analytics/students/{learnerId}/summary`，但 P1-5 仍缺教师端 class summary。教师无法从后端直接查看某门课程下的薄弱知识点、错因分布、资源完成情况和待审核资源积压。

## 目标

- 新增教师端课程学习分析 summary API。
- 返回课程维度的弱知识点、错因分布、资源完成率和待审核资源列表。
- 对教师访问做后端权限检查，教师只能访问自己负责的课程，`admin` 可访问全部课程。
- 不暴露学生原始答案、未审核资源正文、prompt、LLM 输出正文等敏感内容。

## 非目标

- 不新增真实班级、选课、报名或成员表。
- 不新增数据库迁移。
- 不新增前端页面。
- 不引入新依赖。
- 不实现完整 RBAC；临时权限模型继续使用 `X-User-Id`。

## MVP 定义

由于当前没有班级成员模型，本次 MVP 将 `LearningPath.goalId == courseId` 的 learner 集合作为课程学习者集合。该推断只用于 analytics 聚合，不作为长期权限模型。

## 用户价值

- 教师能快速定位课程薄弱知识点。
- 教师能看到常见错因分布，用于补救教学。
- 教师能看到资源完成和审核积压，形成“学习数据 -> 补救资源 -> 审核发布”的闭环证据。
