# 自适应掌握度模型 PRD

## 背景

用户希望继续参考教育 Agent 与自适应学习项目，对当前后端做优化。对比参考项目后，当前后端最薄弱的闭环点是：答题反馈已经能写入 `mastery_record` 并触发路径重规划，但掌握度更新仍使用演示常量，不能真实反映学习者历史状态。

## 目标

- 答题评分后，`beforeMastery` 优先读取该学习者在目标知识点上的最新 `mastery_record`。
- 没有历史记录时使用稳定默认初始掌握度。
- `afterMastery` 按错因类别和当前掌握度做 BKT-lite 更新，避免固定 `0.42 -> 0.58`。
- 保留现有答题提交 API，不新增依赖、不新增数据库表。
- 掌握度变化继续驱动 `LearningPathReplanService`。

## 非目标

- 不引入完整 IRT/BKT 第三方模型。
- 不新增题库管理、rubric 管理或真实 LLM grading。
- 不修改前端页面。
- 不改变 `/api/assessment/answers` 的请求结构。

## 用户价值

- 学生答题反馈能基于自己的历史掌握度，而不是模板数据。
- 路径重规划原因能解释“从多少掌握度变到多少掌握度”。
- 后续做知识图谱、自适应测验、错题推荐时有更可信的 learner state 基础。
