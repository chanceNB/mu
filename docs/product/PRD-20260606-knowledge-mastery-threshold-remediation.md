# Knowledge DAG 掌握度阈值补救优先 PRD

## 背景

当前课程 DAG 路径已经能按 `PREREQUISITE` 依赖排序，并能根据前置知识是否达到完成阈值决定下游节点是否 `LOCKED`。但当多个入口节点同时可学习时，系统还不会优先安排“掌握度低且会阻塞下游知识”的前置知识点，导致补救学习不够明显。

## 目标

- 增加掌握度阈值规则：当前置知识点掌握度低于 `0.6` 时，在可学习节点中优先补救。
- 仅优先处理“有下游依赖”的低掌握度前置知识点。
- 在学习路径节点 `reasonSummary` 中说明补救优先原因。
- 保持已有 `PREREQUISITE` 锁定规则不变。
- 不新增数据库迁移、不新增 DTO 字段、不新增前端页面。

## 非目标

- 不扩展 `learning_path_node` 表字段。
- 不新增 `NEED_REVIEW` 等新状态。
- 不实现资源推荐、测评绑定或预计时长。
- 不实现复杂 BKT/IRT 规划算法。
- 不修改 Assessment 的掌握度更新公式。

## MVP 定义

在课程 DAG 路径生成中，当节点同时满足：

```text
mastery < 0.6
并且该节点是至少一个下游知识点的 PREREQUISITE
并且该节点自身没有未满足前置依赖
```

则该节点在当前可学习节点集合中优先排序。节点状态仍按现有规则输出：`DONE`、`ACTIVE`、`LOCKED`。

## 用户价值

- 学生会先看到真正阻塞后续学习的薄弱前置知识。
- 教师能从路径理由中看到系统为何推荐补救某个知识点。
- 系统从“只按拓扑顺序”推进到“拓扑 + 掌握度”的自适应路径规划。
