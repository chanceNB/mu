# AGENT_RAG_MEMORY.md

## Agent Architecture

- Orchestrator coordinates multi-agent workflows
- Each agent has defined role, tools, and max round limits
- Agent execution writes trace records (traceId, tool calls, model calls)
- AI-generated resources require review status before publish

## RAG Architecture

- Document upload → parse → chunk → embed → index
- Query → retrieve → permission filter → context trim → generate with citations
- RAG answers must include source references
- Retrieval quality should be measurable (evaluation service)

## Core Rules

- Agent Tool → Service layer only
- Max tool round trips enforced
- Model output validated before persistence
- RAG retrieval enforces permission filtering
- Citations visible in frontend (see rag-citation-viewer skill)
