# PROJECT_MEMORY.md

## 1. Project Identity

本项目是"基于大模型的个性化资源生成与学习多智能体系统"。

系统目标是构建一个面向学生个体差异的智能学习平台，实现：

- 对话式学习画像自主构建
- 多智能体协同资源生成
- 个性化学习路径规划
- 资源精准推送
- 智能辅导
- 学习效果评估
- Agent Trace 可审计
- RAG 来源可追溯

## 2. Tech Stack

## 5. Core Rules

- 不允许直接从需求进入代码。
- 开发前必须读取项目记忆。
- 开发前必须进行 Skill Selection。
- 每个需求必须先做 S/M/L Size Classification；小切片走 Fast Lane，中/大型任务走更完整文档流。
- 复杂任务必须判断是否启用 Multi-Expert Subagent；S 小切片默认单 Codex，M/L 按风险启用专家。
- 主 Codex = 总调度；Subagent = 并行分析/设计/审查；编码默认不并行。
- 缺少 Skill 时必须 GitHub 找参考。
- 文档深度按任务体量决定：S 使用 mini TASK + embedded Context Pack + combined Evidence/Acceptance；M 使用 REQ/SPEC/PLAN/TASK/CONTEXT，PRD 按产品行为变化决定；L 使用完整 PRD/REQ/SPEC/PLAN/TASK/CONTEXT。
- 长线父计划使用“父计划 ID + 语义子任务”命名，不再新增人工 `X/Y/Z` 切片名。
- 每次只实现一个 TASK。
- 实现后必须测试、验收、更新文档。
- 前端不能直接调用大模型。
- Agent Tool 不能直接访问 Mapper / Repository。
- 权限不能靠 Prompt。
- RAG 必须有引用来源。
- Agent 必须有 Trace。
- AI 资源必须有审核状态。
